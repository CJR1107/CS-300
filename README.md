# CS 300 – Data Structures and Algorithms Portfolio

## Overview
This repository contains work from CS 300. It includes analysis of run-time and memory usage of data structures (Project One) and a program that sorts and prints course information in alphanumeric order (Project Two).

## Reflection

The main problem I was solving in this course was how to organize and manage course data efficiently. In Project One, I analyzed different data structures to understand their run-time and memory usage. In Project Two, I built a program that reads course data from a file and sorts it in alphanumeric order.

To approach this, I used data structures like vectors, hash tables, and binary search trees. This showed me why data structures are important because they directly impact how fast a program runs and how efficiently it uses memory. Choosing the right one makes a big difference.

One of the biggest challenges I had was getting the program to correctly read and sort the data from the file. I also ran into errors in my code and had trouble making everything work together. I overcame this by testing small parts at a time and fixing issues step by step.

This project changed how I approach building programs. I now spend more time planning before coding and thinking about efficiency instead of just making something work.

It also improved how I write code. I focus more on keeping it clean, readable, and organized so it can be easily updated or expanded later.
